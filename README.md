# AI Dungeon for ALL AI / Simulator Life

`Simulator Life` - это личная библиотека системных промптов, аватаров, навыков и сценарных заготовок для текстовых AI-RPG. Репозиторий рассчитан на запуск симуляции в разных моделях: ChatGPT, Claude, Gemini, DeepSeek, Grok и других LLM, которые поддерживают длинный контекст и системные инструкции.

Главная идея проекта: дать модели не просто роль "ведущего", а устойчивый игровой режим с каноном, памятью недавних событий, персонажами, навыками, стартовыми сценами и правилами поведения симулятора.

## Что внутри

| Путь | Назначение |
|------|------------|
| `System/` | Системные промпты симулятора и AIGM. Здесь лежат JSON-версии `prompt_simulator_v3-v10` и Markdown-порты `prompt_simulator_v11_*` под разные модели. |
| `Avatars/` | Более компактная подборка аватаров, вынесенных в отдельную папку. |
| `Skills/` | Навыки, дары и игровые механики, которые можно подключать к аватарам или сценариям. |
| `*.txt`, `*.json` в корне | Основная коллекция аватаров, экспериментов, сценариев и старых форматов. |
| `.codex/skills/story-creator/` | Codex skill для создания, переписывания и канон-согласованного улучшения контента проекта. |

## Быстрый старт

1. Выбери системный промпт из `System/`.
2. Вставь его в AI как system prompt, developer prompt или первое сообщение, в зависимости от интерфейса модели.
3. Выбери аватар из корня репозитория или `Avatars/`.
4. При необходимости добавь навык из `Skills/`.
5. Запусти сцену: отправь модели аватар, стартовые условия и коротко укажи желаемый тон игры.

Для новых запусков лучше начинать с актуальных `prompt_simulator_v11_*`, если целевая модель поддерживает длинные Markdown-инструкции. Старые JSON-версии полезны для совместимости, сравнения поведения и восстановления прежних режимов.

## Как выбирать системный промпт

- `prompt_simulator_v11_opus.md` - порт под Claude/Opus-подобные модели.
- `prompt_simulator_v11_gemini.md` - порт под Gemini-подобные модели.
- `prompt_simulator_v11_deepseek.md` - порт под DeepSeek-подобные модели.
- `prompt_simulator_v11_grok420.md` - порт под Grok-подобные модели.
- `prompt_simulator_v10*.json` и старше - исторические JSON-версии и альтернативные ветки.
- `prompt_aigm.json` - отдельная AIGM-линия для режима AI game master.

Если не уверен, какой файл брать, выбирай порт под конкретную модель. Если такого порта нет, начинай с самой свежей общей версии и адаптируй формат под ограничения модели.

## Codex Skill: Story Creator

В репозитории есть встроенный Codex skill:

`.codex/skills/story-creator/`

Он помогает:

- создавать новые аватары, NPC, миры, сценарии и эксперименты в стиле проекта;
- улучшать существующие файлы без поломки канона;
- находить ближайшие живые эталоны перед правкой;
- не копировать навыки целиком, когда достаточно summary и ссылки;
- портировать `prompt_simulator_v*` под другие модели без потери инвариантов;
- проверять anti-patterns, структуру, именование и diff перед сохранением.

Чтобы использовать skill локально в Codex, синхронизируй папку:

```powershell
$target = "$env:USERPROFILE\.codex\skills\story-creator"
New-Item -ItemType Directory -Force -Path $target | Out-Null
Copy-Item -Recurse -Force ".codex\skills\story-creator\*" $target
```

После этого в запросах можно писать, например: `Используй story-creator и улучши этот аватар`.

## Рекомендации по работе с контентом

- Не создавай новый файл "по памяти", если в проекте уже есть похожий формат.
- Для аватаров не дописывай характер и мотивацию без явного основания в запросе или каноне.
- Для миров важна играбельность: правила, ограничения, конфликты и точки входа в сцену.
- Для NPC важна функция в сцене, а не только биография.
- Для навыков обязательно нужны ограничения, условия применения и понятная игровая роль.
- Для `prompt_simulator_v*` сначала найди реально последнюю версию в `System/`, затем вноси изменения.
